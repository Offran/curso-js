Variáveis JS

Falando sobre:
var
let
const

var:
Passou a ser menos usada após 2015 por bons programadores, por sua menor especificação em comparação às outras, podendo criar pequenos erros semânticos, mas não de programação.

let:
Diferente de var, o let tem escopo de bloco, instrução ou expressão. Não são globais ou de escopo de função.

  1. Diferente ed var, elas tem um escopo de bloco, instrução ou expressão. Não são globais ou escopo de função. O escopo bloco { let x = 5;} não pode ser acessado fora dele, então se um escopo for criado, o let de dentro dele não será validado.
  2. Não podem ser redeclaradas. Por exemplo: declarar "let x = 5;" e mais além declarar "let x = 10;" (a não ser que saia do escopo)
  3. Devem ser declaradas antes do uso.

const:
São valores constantes e portanto inalterados.
const price1 = 5;
const price2 = 6;
let total = price1 + price2;
document.getElementById("demo").innerHTML = "The total is: " + total;

Nota: Como pode ser visto, também é possível concatenar uma string com um number, contudo isso transforma o number em string.

Quando usar o const?
nova matriz, novo objeto, nova função, novo RegExp (serão melhor abordados mais pra frente).

Quando usar var, let ou const?
1. Sempre declare variáveis
2. Sempre use const se o valor não deve ser alterado
3. Sempre use constse o tipo não deve ser alterado (Arrays e Objetos)
4. Só use letse não puder usarconst
5. Use apenas varse você DEVE oferecer suporte a navegadores antigos.

Declarando uma variável vazia:
var carName;
Com isso a declaração é automaticamente "undefined", sendo preciso colocar o siinal de igual (=) para atribuir algum valor.

Nota: Embora não seja obrigatório terminar com o ponto e vírgula, ele é altamente recomendado, assim reforçando a questão da clareza e limpeza do código. No caso de fazer em uma única linha, apenas uma vírgula é necessária na declaração, por exemplo: let x = 5, y = 2, z = 3;

Hoisting

É o comportamento padrão de JS de mover as declarações para o topo de seu escopo antes da execução do código.
